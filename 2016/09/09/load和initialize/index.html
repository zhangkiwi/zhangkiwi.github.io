+load和+initialize

Sep 9, 2016

NSObject 类中有两个非常特殊的类方法 +load 和 +initialize，这两个方法有让人困惑的地方。本文结合网上的一些结论从源码角度分析它们的区别和共同点。本文所使用的源码。

+load

+load方法网上总结。

  1. 每当将类或分类添加到Objective-C运行时时调用
  2. 其父类方法的调用在子类之前
  3. 分类方法的调用在本类之后
  4. 如果两个库有依赖关系，则被依赖的库中的+load方法优先调用。

子类、父类和分类中的 +load 方法的实现是被区别对待的。就是说如果子类没有实现 +load 方法，那么当它被加载时 runtime 是不会去调用父类的 +load 方法的。同理，当一个类和它的分类都实现了 +load 方法时，两个方法都会被调用。因此，我们常常可以利用这个特性做方法混淆（Method Swizzling）。

+initialize

  1. +initialize 在收到第一条消息之前调用。
  2. 父类的调用在子类之前。
  3. 如果子类没有实现 +initialize 方法，那么继承自父类的实现会被调用；如果一个类的分类实现了 +initialize 方法，那么就会对这个类中的实现造成覆盖。

第一条+initialize 在收到第一条消息之前调用，可以从苹果官方文档中找出答案。只说是收到第一条消息之前调用，消息包括实例方法和类方法的调用。 +initialize 方法是以懒加载的方式被调用的，如果程序一直没有给某个类或它的子类发送消息，那么这个类的 +initialize 方法是永远不会被调用的。这样可以节省系统资源，避免浪费。
