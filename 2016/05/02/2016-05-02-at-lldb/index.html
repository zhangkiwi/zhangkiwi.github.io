输出命令

输出命令是我们调试过程中用得最多的命令，在LLDB中有多种输出命令，
p,po,print,call。我们实际看看它们的输出有什么不同。

上面的输出命令几乎没什么不同，那我们来看看各种命令的原理是什么，用LLDB的help命令查看一下。

可以看到它们都是expression命令的别名，但是po是输出对象类型的别名。如果输出的变量不是对象类型就可以看出结果。

可以看到po命令只是输出数值，而p，print命令则输出了变量的类型。并且p命令还可以指定打印格式。

观察点命令

上面的breakpoint 是对方法生效的断点，还有一个是对地址生效的断点，watchpoint，用它来监听属性值的变化是最好不过的，虽然我们也可以对属性进行断点，如果只是普通赋值而没有调用setter方法的时候就应该用到watchpoint命令了。

watchpoint set variable

watchpoint set variable 传入的时候是变量名，不能用方法 所以如果这里用watchpoint set variable self.num 是不行的。应为self.num调用的是setter方法进行的赋值。

expression

这个命令用来执行一个命令，并将表达式的结果输出。
他有两个作用一是：动态改变程序的运行轨迹，如我们想在程序运行时改变self.view的背景色。

上面的第一个命令用来改变颜色，第二个命令用来刷新UI。它的这个动态特性还可以给它动态的添加一个变量，如下：

开始我们动态的添加了一个test（必须用$）变量赋值为12，然后打印了test的值，可以看到会有一个$符，我们每次查询的结果会保存在一些持续变量中($[0-9]+)，这样我们可以在后面的查询中直接使用这些值。比如上面的命令 expression $2 + $3 最后输出的结果是 17.
