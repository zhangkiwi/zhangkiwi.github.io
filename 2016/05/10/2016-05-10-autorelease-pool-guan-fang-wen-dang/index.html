在这个方法中执行具体的自动释放对象的加入操作，它分三种情况考虑：

  • 当前 page 存在且没有满时，直接将对象添加到当前 page 中，即 next 指向的位置；
  • 当前 page 存在且已满时，创建一个新的 page ，并将对象添加到新创建的 page 中；
  • 当前 page 不存在时，即还没有 page 时，创建第一个 page ，并将对象添加到新创建的 page 中。

现在我们理一下自动释放池push的调用栈，

从objc_autoreleasePoolPush() –> AutoreleasePoolPage::push() –> autoreleaseFast(id obj) –> add(obj)

AutoreleasePoolPage

搜索AutoreleasePoolPage可以找到一个AutoreleasePoolPage的类，这个类的结构如下，分析得出这是一个双向链表的节点。

其主要属性的作用：

  • next 指向最后一个添加的自动释放对象的下一个地址
  • thread 当前线程，自动释放池与线程一一对应
  • parent 指向父节点
  • child 指向子节点

POOL_SENTINEL（哨兵对象）

POOL_SENTINEL只是nil的宏定义。

在每个自动释放池初始化调用objc_autoreleasePoolPush的时候都会把一个POOL_SENTINEL push到自动释放池的栈顶，并且返回这个POOL_SENTINEL对象。
而当方法objc_autoreleasePoolPop调用时，就会向自动释放池中的对象发送release消息，直到第一个POOL_SENTINEL，并且回移Next指针到正确位置。

Autorelease Pool 使用

根据苹果官方文档中描述，我们知道在下面三种情况下是需要我们手动添加 autoreleasepool 的：

  • 如果你编写的程序不是基于 UI 框架的，比如说命令行工具；
  • 如果你编写的循环中创建了大量的临时对象；
  • 如果你创建了一个辅助线程。
